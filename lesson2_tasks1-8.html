1. Дан код:
var a = 1, b = 1, c, d;
c = ++a; alert(c); // 2
// c = к переменной а прибавляем единицу
d = b++; alert(d); // 1
// d = выводим предыдущее значение b, затем прибавляем единицу
c = (2+ ++a); alert(c); // 5
// с = 2 + 3, т.е. 2 + (a = a + 1), а т.к. в первом примере тоже в переменную a прибавляли единицу, получается a = 3
d = (2+ b++); alert(d); // 4
// d = 2 + 3, но выводим сумму с предыдущим значение b, т.е. 2 + 2, т.к. во втором примере в переменную b прибавляли единицу, затем к переменной b прибавляем еще единицу, получается b = 3
alert(a); // 3
alert(b); // 3
Почему код даёт именно такие результаты?
Ответ:
В JavaScript есть префиксная и постфиксная форма записи операторов инкрементрирования и декрементирования.
В постфиксной форме сначала происходит возвращение значения, а потом выполняется инкрементирование.
В префиксной форме инкрементирование производится сразу, а возврат — уже с обновленным значением.

2. Чему будет равен x в примере ниже?
var a = 2;
var x = 1 + (a *= 2);
Ответ: 5.
Оператор *= это сокращенный вид оператора присвоения переменной значения самой переменной, умноженной на заданное число.
Развернутая запись заданного примера: 1 + (a = a * 2 ).
